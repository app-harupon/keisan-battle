# ケイサンバトル 公開ロードマップ

「GitHubで公開 → 多くの人に遊んでもらう → App Store / Google Play で配信」までの計画。

---

## フェーズ1: GitHub + Web公開(いま)

**目的**: コードを管理し、URLひとつで誰でも遊べる状態にする。

- [x] Gitリポジトリ化・GitHubへプッシュ
- [x] GitHub Pages で公開(`https://<ユーザー名>.github.io/keisan-battle/`)
- [ ] URLを友達・SNSでシェアして遊んでもらう

**ポイント**: 単一HTMLなのでビルド不要。`main` にプッシュするだけで公開版が更新される。

## フェーズ2: フィードバック収集と品質向上

**目的**: 実際に遊んでもらい、ゲームとして磨く。ストア審査に耐える完成度にする。

- [ ] GitHub Issues でバグ報告・要望を受け付ける(Issueテンプレート整備)
- [ ] レベルモード進行・ハイスコアの永続化(localStorage)
- [x] 漢字語彙データの拡充(全10レベル・約720語、Lv3以上は意味つき)→ さらなる拡充・漢検リストとの精査は継続
- [x] 効果音とバトル演出(READY GO・ゴング・攻撃/命中/勝敗SE、ミス時の解説タイム、ふりかえり機能)
- [ ] BGMの追加
- [x] 遊んだ人の反応をもとに難易度バランス調整(計算バトルのペースを全体的に減速、漢字Lv7以降に難読漢字を追加)
- [x] 正解・不正解エフェクト、ダメージ効果音の変更
- [x] 同じ問題が連続で出ない仕組み(連続出題防止)
- [x] ミス問題の復習モード(教科ごとに間違えた問題をlocalStorageへ蓄積し再挑戦できる)
- [x] 一時停止ボタン(つづける/ホームに戻るの2択。敵の攻撃間隔やサバイバルの残り時間を正確に引き継いで再開)
- [x] 入力スピード設定を5段階化、選択中のボタンが色で分かるように修正
- [ ] (任意)簡易アクセス解析でどのモードが遊ばれているか把握

## フェーズ2.5: 大型機能(オンライン対戦・プロフィール・新教科・ホーム画面再編)

**目的**: CPU対戦だけでなく人と対戦できるようにし、やり込み要素(称号・バッジ)と学べる教科を広げる。
UI・ナビゲーションは実装済み、オンライン対戦本体と称号・バッジの獲得条件は今後あらためて行う。

- [x] 画面構成を「ホーム → 教科をえらぶ → モードをえらぶ → レベルをえらぶ → バトル」の階層に再編。設定・プロフィールはホーム画面のタイルから開ける形に変更
- [x] 教科をデータ駆動化(`SUBJECTS` 配列)し、れきし を実装(かがく・えいご・おんがく・びじゅつは「準備中」表示のまま)
- [x] れきしモード: `history-data.js` に時代順のクイズデータを追加。「〜を行ったのは…と誰?」形式の四択クイズ専用エンジン(選択肢シャッフル・キーボード1〜4対応)で動作(初期実装ではかんじと同じ読み方エンジンを流用していたが、四択クイズ形式に刷新)
- [x] れきしを「にほんし」「せかいし」の2教科に分割。`world-history-data.js` を新設し、古代文明〜現代までの世界史クイズ(全10レベル・100問)を追加。ミス問題バンクも教科ごとに独立して管理される
- [x] オンライン対戦タイルを追加(「準備中」表示。押すとトースト通知のみ)
- [x] プロフィール機能の土台(👤ボタン → プレイヤー名の設定・保存)
- [x] 称号・バッジのカタログ(`TITLE_CATALOG` / `BADGE_CATALOG`)とプロフィール画面での表示(すべて未解放のプレースホルダー)
- [x] 「ちょうどいいバトル」を「診断バトル」に刷新: HP制のバトルではなく全12問のスコアアタック形式に変更。自分の実力を数値で確認でき、今ちょうどいいレベルを調べる用途にも使える
- [ ] オンライン対戦の実装: GitHub Pagesは静的配信のみのため、別途リアルタイム通信用のサーバー(WebSocketの中継やPartyKit/Supabase Realtimeなど)が必要。攻撃処理は「自分の入力」「相手の入力」に分離済みで拡張しやすい
- [ ] 称号・バッジの獲得条件の実装(初勝利・5連勝・全ステージ制覇など、`BADGE_CATALOG` のidに対応する判定ロジック)
- [ ] 新教科(かがく・えいご・おんがく・びじゅつ)の問題データ作成(`kanji-data.js` / `history-data.js` / `world-history-data.js` と同様の形式を教科ごとに用意する想定)
- [ ] プロフィール・バッジのオンライン対戦との連携(対戦相手のプロフィール表示など)

## フェーズ3: PWA化(ストアへの橋渡し)

**目的**: 「ホーム画面に追加してアプリのように遊べる」状態にする。ストア配信の土台にもなる。

- [ ] `manifest.json` 追加(アプリ名・アイコン・スタンドアロン表示)
- [ ] Service Worker 追加(オフラインでも遊べるようにキャッシュ)
- [ ] アイコン画像の作成(512x512 ほか各サイズ)

## フェーズ4: ネイティブアプリ化(Capacitor)

**目的**: 今のWebコードをほぼそのまま iOS / Android アプリとしてパッケージする。

- [ ] Node.js プロジェクト化して [Capacitor](https://capacitorjs.com/) を導入
- [ ] Android ビルド(Android Studio が必要 / Windowsで可)
- [ ] iOS ビルド(**Mac + Xcode が必須**。手元にMacがない場合はクラウドMacサービス等を検討)
- [ ] 実機テスト

**補足**: コードは単一HTMLのままでよく、Capacitorが「Webページを表示するだけのネイティブアプリ」に包んでくれる。オンライン対戦を入れる場合もWeb側の実装がそのまま使える。

## フェーズ5: ストア公開

### Google Play(先にこちらが楽)

- [ ] Google Play デベロッパーアカウント登録(**初回のみ $25**)
- [ ] ストア掲載素材(アイコン、スクリーンショット、説明文、プライバシーポリシーURL)
- [ ] 内部テスト → クローズドテスト(Googleの要件: 一定人数のテスターで一定期間のテストが必要)→ 製品版公開

### App Store

- [ ] Apple Developer Program 登録(**年額 $99**)
- [ ] App Store Connect でアプリ登録・素材アップロード
- [ ] 審査対応(「Webサイトをただ包んだだけ」は却下されやすいため、PWA機能+ネイティブらしい体験=オフライン動作・ハプティクス等を入れておく)

### 共通で必要なもの

- プライバシーポリシーのページ(GitHub Pages上に置けばOK)
- 対象年齢・コンテンツレーティングの申告(教育系・全年齢なので問題になりにくい)

## 判断ポイント

- **フェーズ2→3の移行タイミング**: Webで遊んでくれる人が付き、直したいバグが落ち着いたら
- **iOSを本当にやるか**: Mac環境のコストが掛かるため、まずは Google Play のみ先行も現実的
- **オンライン対戦**: ストア公開とは独立して進められる。入れるならサーバー費用(無料枠のあるホスティングで開始可)を検討
